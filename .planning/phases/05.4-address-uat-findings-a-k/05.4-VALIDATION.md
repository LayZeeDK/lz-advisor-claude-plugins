# Phase 05.4 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | `claude -p` (non-interactive Claude Code skill invocation) |
| **Config file** | none — invocations use `--plugin-dir plugins/lz-advisor` + `--verbose` |
| **Quick run command** | `claude --model sonnet --effort medium --plugin-dir plugins/lz-advisor -p "/lz-advisor.<skill> <task>" --verbose` |
| **Full suite command** | 6-session UAT pipeline script (D-03 Stage 2) — see RESEARCH.md |
| **Estimated runtime** | ~30-60s per smoke test; ~15-20min for 6-session pipeline |

---

## Sampling Rate

- **After every task commit:** Run the `claude -p` smoke test for the finding the task closes
- **After every plan wave:** Run all smoke tests for findings touched by the wave
- **Before `/gsd-verify-work`:** All 10 per-finding smoke tests green AND 6-session UAT pipeline produces acceptable first-try/post-fix metrics matching Phase 5.3's 57% / >=80% gate
- **Max feedback latency:** ~60 seconds for per-task verify
