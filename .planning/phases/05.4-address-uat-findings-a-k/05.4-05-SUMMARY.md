## Accomplishments

- **E-runtime (D-16) closure at prompt layer.** advisor.md Edge Cases first paragraph replaced with an imperative contract mandating the literal `Assuming X (unverified), do Y. Verify X before acting.` sentence frame, forbidding paraphrase of any of the fixed words (`Assuming`, `(unverified)`, `do`, `Verify`, `before acting.`), naming the clarification request as a dead-end with no follow-up turn, and anchoring the pattern with two illustrative examples.
- **F-runtime (D-17) closure at prompt layer.** reviewer.md and security-reviewer.md Output Constraint sections each prepended with an imperative contract: the response MUST begin with the literal `### Findings` header and MUST include the literal `### Cross-Cutting Patterns` (reviewer) / `### Threat Patterns` (security-reviewer) header, with paraphrasing, bolding, and translation of the headers explicitly forbidden. Added MANDATORY-even-when-short escape hatch: when the synthesis section has nothing substantive, emit the header followed by a one-sentence explanation.
- **F-runtime (D-17) closure at orchestration layer.** lz-advisor.review/SKILL.md and lz-advisor.security-review/SKILL.md Phase 3 sections rewritten from severity-grouping reformatters to verbatim pass-through stages. The skill prepends a one-line `## Review Summary` / `## Security Review Summary` scope line, then emits the reviewer/security-reviewer response unchanged so the `### Findings` + synthesis-section headers reach stdout. An explicit `Do NOT` list enumerates each old reformatting vector (severity group, header bold, wrapper section, Recommended Action, attribution tag).
- **A-smoke-script-bug closure.** HIA-discipline.sh now captures `REPO_ROOT="$(git rev-parse --show-toplevel)"` at line 11 -- before `cd "$SCRATCH"` at line 15. The buggy line 37 `git -C "$(git rev-parse --show-toplevel)" grep ...` (which resolved `show-toplevel` from inside the scratch repo and returned the wrong directory) now reads `git -C "$REPO_ROOT" grep ...`, fixing the ambiguous-argument failure on the Finding A static check.
- **Sibling script scan confirmed clean.** J-narrative-isolation.sh, KCB-economics.sh, and DEF-response-structure.sh all capture `PLUGIN_DIR` from `git rev-parse --show-toplevel` ABOVE their respective `cd "$SCRATCH"` lines and never re-invoke `git rev-parse --show-toplevel` after cd. No sibling patch needed; the proactive scan surfaced no second occurrence of the anti-pattern.
