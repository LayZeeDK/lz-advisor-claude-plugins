- If the user asked to review "recent changes" or "my changes", use `git diff` or `git log` to identify changed files, then read them

- Issues a linter or type checker would catch (formatting, unused imports, type errors)
- Style preferences or subjective suggestions
- Pre-existing issues outside the review scope
- Pedantic nitpicks

- Concrete fix suggestion (per D-16)

- A strengths or positive observations section (per D-18)
- A "Recommended Action" or next-steps section (per D-19)
- Opus attribution tags or a separate "Strategic Direction" section (per D-17)

If the reviewer identified cross-cutting patterns (per D-11), weave them into the findings naturally -- for example, note the shared root cause when presenting related findings.

After the output section closing tag, add a brief closing line:

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User code -> reviewer agent prompt | Untrusted code snippets from the reviewed codebase are embedded in the Opus reviewer's prompt |
| User request -> skill execution | User-controlled review scope (file paths, directories) determines what executor reads |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing | agents/reviewer.md | accept | Agent is invoked only by skill via allowed-tools; no direct user invocation path. Low risk -- skill-orchestrated only. |
| T-04-02 | Tampering | skills/lz-advisor-review/SKILL.md | accept | Skill is a read-only Markdown file; no runtime mutation. Plugin installation is user-initiated. |
| T-04-03 | Information Disclosure | agents/reviewer.md | mitigate | Reviewed code snippets are sent to Opus via Agent tool (same trust boundary as normal Claude Code usage). Skill instructs executor to summarize findings, not dump entire files. Agent has Read/Glob tools but maxTurns: 1 limits data exfiltration surface. |
| T-04-04 | Elevation of Privilege | agents/reviewer.md | mitigate | Agent tools restricted to ["Read", "Glob"] -- no Write, Edit, Bash, or other mutation tools. Agent cannot modify files or execute commands. maxTurns: 1 prevents multi-step escalation. |
| T-04-05 | Injection | skills/lz-advisor-review/SKILL.md | accept | Malicious code under review could contain prompt injection attempts in comments/strings. Risk is inherent to any code review tool (human or AI). Opus model has built-in prompt injection resistance. No additional mitigation warranted for v1. |
</threat_model>
