# Phase 05.3 -- Validation Strategy

> Per-phase validation contract for feedback sampling during execution.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | `claude -p` + NDJSON `jq` parsing (no unit test framework; validation is empirical A/B UAT) |
| **Config file** | None -- UAT is imperative scripts run by spawned subagents |
| **Quick run command** | `claude --model sonnet --effort medium --plugin-dir plugins/lz-advisor -p "/lz-advisor.<skill> <prompt>" --verbose --output-format stream-json` |
| **Full suite command** | Orchestrator spawn of 8 parallel UAT subagents + 1 grader (run per pass) |
| **Estimated runtime** | ~15-20 min per pass (8 subagents in parallel, grader serial); ~40 min total for both passes |

The plugin is pure Markdown/YAML with no compiled code; nothing to unit-test. All validation is A/B UAT on real Claude Code runs plus static grep assertions on the edited files.

---

## Sampling Rate

- **After every task commit:** No automated test suite runs on file changes. Static grep assertions (e.g., `git grep -c "Context Trust Contract"`) can be spot-checked locally; not gated.
- **After every plan wave:** Plan 3 single-UAT validation run (one skill, one pass) before Plan 4's full 8-subagent matrix -- catches `Agent(isolation: "worktree")` and `--plugin-dir <worktree>` resolution bugs before scaling up.
- **Before `/gsd-verify-work`:** Full 8-subagent UAT + grader produces `05.3-UAT-RESULTS.md`. D-04 dual gate (>=80% first-try success AND <=4 tool calls average) must pass green.
- **Max feedback latency:** ~20 min (single pass); ~40 min (both passes).

## A/B Measurement Methodology

For each of the 4 skills, the grader compares baseline-pass metrics against post-fix-pass metrics:

| Metric | Baseline Source | Post-fix Source | Expected Direction |
|--------|-----------------|-----------------|--------------------|
| First-try success rate | 05.2-FIELD-TEST.md 4/7 = 57% + 4 baseline runs this phase | 4 post-fix runs | `>=80%` target |
| Advisor tool calls per consultation | 05.2-FIELD-TEST.md table + 4 baseline runs | 4 post-fix runs | `<=4` avg |
| maxTurns exhaustion rate | 05.2-FIELD-TEST.md 3/7 = 43% | 4 post-fix runs | lower |
| Advisor word-count cap (<=100w) | Baseline captured this phase | 4 post-fix runs | stable or better |
| Reviewer/security tone (UAT 4 rubric) | 2 baseline runs | 2 post-fix runs | stable or better |

**Primary gate (D-04):** Both (a) and (b) must pass. Grader fails the gate if either is missed.

**Secondary / information-only:** maxTurns exhaustion, word-count cap, UAT 4 tone. Regressions here are flagged but do not individually fail the gate.

**Cross-skill regression guard:** All 4 skills are in the UAT harness. Any skill that was passing baseline and drops post-fix triggers a regression flag even if the overall gate passes. Don't accept "overall better, some worse."
