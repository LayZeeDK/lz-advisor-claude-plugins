# Phase 05.3 Plan 02: Shared Context-Packaging Reference Summary

Extracted inline advisor-prompt packaging prose from all four SKILL.md files into a new `plugins/lz-advisor/references/context-packaging.md` shared reference (Common Contract + Proposal template + Verification template + worked examples + decision table), and added D-13 orientation-budget guidance to plan and execute orient phases.

## What Was Built

### 1. New reference file: `plugins/lz-advisor/references/context-packaging.md`

Four top-level sections:

1. **Common Contract** -- four rules every consultation follows (inline file contents, targeted request, enumerated/tagged structure, turn-budget awareness), plus a Source Material vs Your Own Findings sub-rule preserving the D-05 verbatim-vs-summary distinction.
2. **Proposal Consultation Template** -- structure block + worked example (Compodoc / Storybook 10.3.5 setup) + Short-Form Variant callout for mid-execute re-consultations.
3. **Verification Consultation Template** -- structure block + worked example adapted from the successful Phase 5.2 security-review field-test pattern (Compodoc dependency setup, OWASP A06 framing, Threat Model Context block).
4. **When to Use Each Template** decision table mapping each of the 6 call sites across the 4 skills to a template.

File size: 8,189 bytes (well under the ~5K-token compaction ceiling; roughly 2x advisor-timing.md). ASCII-only.

### 2. SKILL.md edits (all four skills)

All four SKILL.md files now @-load `context-packaging.md` immediately below the existing `advisor-timing.md` @-load, using the same `@${CLAUDE_PLUGIN_ROOT}/references/context-packaging.md` pattern.

The inline packaging prose was removed and replaced with one-line template pointers per the SKILL/call-site -> template map below.

## D-13 Orientation-Budget Bullet

Added to the orient phase of both `lz-advisor.plan` and `lz-advisor.execute` (the two skills with substantive orientation workflows):

> Stop exploring when you have enough context to formulate a specific question for the advisor. Do not read bundled, minified, or files under `node_modules/*/dist/`.

Not added to review or security-review skills: their orientation is scoped to `<scan>` with a CLAUDE.md-reading step, and D-13's concern (over-investigation into node_modules internals during plan/execute orientation) does not apply to review workflows that target user-specified files.
