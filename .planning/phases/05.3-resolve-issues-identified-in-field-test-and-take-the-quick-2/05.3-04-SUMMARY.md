## Deviations

1. **Serial instead of parallel.** Plan 04's original `Agent(isolation: "worktree")` parallel-spawn variant was not used. Per Plan 03 VALIDATION-RUN.md A2 verdict (PARTIAL), the serial orchestrator-managed pattern was adopted as the primary execution path. This variant was empirically validated by Plan 03's single-UAT run. No change to result interpretability.
2. **Orchestrator-driven instead of subagent-driven.** The 8 UATs ran from the orchestrator session rather than from 9 spawned subagents (8 UAT runners + 1 grader) because: (a) earlier `gsd-executor` attempts for this phase stalled on the 600s watchdog, and (b) `claude -p` subagent invocations in `Agent(isolation: "worktree")` showed timing problems during Plan 03. Outputs are equivalent; only the harness runner identity differs. User approved this mode explicitly via AskUserQuestion ("Orchestrator-driven serial runs") before execution.
3. **Grading done inline by orchestrator** (not by a spawned grader subagent). Same rationale as above. The grader-prompt.md rubric (UAT 4 Likert, D-04 verdict logic, per-UAT verdict rules, model-version integrity check) was applied verbatim.
4. **Two cleanup commits removed from ngx-smart-components.** The UAT 4 (execute/baseline) run's `execute` skill implemented the Compodoc changes and committed to its detached-HEAD scratch worktree. An earlier pre-phase test session had also created stray commits (`c3d9d0a`, `8696de7`) on ngx-smart-components main. User reset main back, and orchestrator removed the scratch-worktree reflog entries + ran `git gc --prune=now` so those SHAs are now unreachable and purged from the object database. ngx-smart-components main is at `019a26a` clean.
