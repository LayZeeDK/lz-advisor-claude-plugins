# Phase 5.2 UAT Gap Research: Advisor Agent Response Mechanics

**Researched:** 2026-04-14
**Domain:** Claude Code agent loop, maxTurns behavior, subagent response flow, preamble elimination
**Confidence:** HIGH (confirmed against official Anthropic documentation)
**Triggered by:** 05.2-UAT.md Test 2 failure -- advisor preamble persists despite effort: medium

## Summary

The advisor agent's preamble ("Let me verify the executor schema and current config.") persists across phases 5.1 and 5.2 because the root cause is structural, not behavioral. With `maxTurns: 1`, the agent is terminated via `error_max_turns` after its first tool-use turn, before it can produce a post-tool synthesis response. The pre-tool-call text (preamble) becomes the only "Response" returned to the parent session. `effort: medium` was a misguided fix -- effort controls reasoning depth, not output format.

Fix: `maxTurns: 3` + `effort: high`. With maxTurns > 1, internal preamble is invisible to the parent (only the final text-only response returns). Restoring effort: high gives the Opus advisor deeper strategic reasoning.

## Agent Loop Mechanics

**Source:** [How the agent loop works](https://platform.claude.com/docs/en/agent-sdk/agent-loop) (official Anthropic docs)

### Turn counting

A turn is one round trip: Claude produces output with tool calls, tools execute, results feed back. `maxTurns` counts **tool-use turns only**. The final text-only response is free (not counted).

Loop pseudocode:
```
while true:
  response = claude_next_response()
  if response has NO tool calls:
    return success(response.text)       # parent sees this

  tool_use_count += 1
  execute_tools(response.tool_calls)
  feed_results_back()

  if tool_use_count >= maxTurns:
    return error_max_turns()            # result field NOT available
```

Key quote from official docs:
> "You can cap the loop with max_turns / maxTurns, which counts tool-use turns only. For example, max_turns=2 in the loop above would have stopped before the edit step."

### What the parent receives

From [Create custom subagents](https://code.claude.com/docs/en/sub-agents):
> "Only the subagent's final text response comes back to your context, plus a small metadata trailer with token counts and duration."

- On `success`: parent receives the final text-only response (the synthesis)
- On `error_max_turns`: `result` field is NOT present; parent receives the last AssistantMessage text (the preamble)

### Why maxTurns: 1 fails

With `maxTurns: 1` and tools available:
1. Agent writes "Let me verify..." + issues Read/Glob (1 tool-use turn)
2. Tools execute, results feed back
3. `tool_use_count (1) >= maxTurns (1)` -- STOP with `error_max_turns`
4. Agent never gets turn 2 to synthesize tool results
5. Parent receives pre-tool-call preamble as the "Response"

### Why maxTurns: 3 fixes it

With `maxTurns: 3`:
1. Turn 1 (tool-use, count 1/3): Agent issues Read/Glob -- results back
2. Turn 2 (optional, count 2/3): Agent issues more reads if staged -- results back
3. Agent writes text-only synthesis -- loop ends with `success`
4. Parent receives ONLY the synthesis; all internal preamble is invisible

The buffer turn (3 vs 2) handles staged verification: Glob to discover files (turn 1), then Read specific files (turn 2), then synthesize (free text-only).

### Effort parameter

From [Agent loop docs](https://platform.claude.com/docs/en/agent-sdk/agent-loop):

| Level | Behavior | Good for |
|-------|----------|----------|
| low | Minimal reasoning, fast responses | File lookups, listing directories |
| medium | Balanced reasoning | Routine edits, standard tasks |
| high | Thorough analysis | Refactors, debugging |
| max | Maximum reasoning depth | Multi-step problems requiring deep analysis |

Effort controls **internal reasoning depth**, not output length. Output length is governed by the system prompt ("under 100 words"). The advisor's effort: medium was a misguided attempt to reduce preamble by reducing thinking -- it had no effect on the structural maxTurns issue.

## Preamble Elimination Prompt Patterns

**Source:** Ralph Loop cybernetics research at `D:\projects\github\LayZeeDK\lz-cybernetics-ai-plugins\research\ralph-loop\`

These patterns reduce wasted tokens in internal agent turns (optimization, not the structural fix).

### Action verb selection (BEST-PRACTICES.md, IMPLEMENTATION.md)

| Instead of | Use | Why |
|------------|-----|-----|
| "Read the file" | "Study the file" | Triggers analysis, not narration |
| "Check the config" | "Verify the config" | Implies validation, not description of intent |
| "Look at" | "Examine" / "Analyze" | Structured findings, not commentary |

### Anti-deferral language (IMPLEMENTATION.md lines 290-306)

| Pattern | Effect |
|---------|--------|
| "don't assume not implemented" | Forces code search, prevents reimplementation preamble |
| "resolve them or document them" | Forces closure, eliminates "let's discuss" preamble |
| "if functionality is missing then it's your job to add it" | Prevents deferral narration |

### Black box verification principle (PLUGIN-GUIDE.md)

> "Treat thinking tokens as opaque black-box output, not as reliable internal state. Any control architecture that depends on the agent's self-reported 'understanding' or 'confidence' is cybernetically unsound."

This validates the structural fix (maxTurns: 3) over prompt-only approaches. The mechanism (enough turns to reach text-only response) is more reliable than trying to control internal narration.

### 9s numbering convention for guardrails (IMPLEMENTATION.md lines 690-710)

Critical instructions get higher numbers and sort to the bottom of the prompt where they function as invariants. The advisor's "Final Response Discipline" section is already positioned at the end, which aligns with this pattern.

## Fix Specification

### advisor.md changes

| Field | Current | Target | Rationale |
|-------|---------|--------|-----------|
| `maxTurns` | 1 | 3 | Structural fix: allow tool use + synthesis + buffer |
| `effort` | medium | high | Restore: medium was misguided preamble workaround |

### Optional prompt refinements (advisor.md body text)

1. Replace "When using Read or Glob to verify claims, issue tool calls without narration" with language that acknowledges internal turns are invisible: "Your tool calls and intermediate text are internal. Only your final text-only response reaches the executor."
2. Apply Ralph Loop action verbs: "Study" not "Read", "Verify" not "Check"
3. Simplify Final Response Discipline -- the structural fix makes some instructions redundant

- `model: opus` -- unchanged
- reviewer.md, security-reviewer.md -- these agents have different maxTurns/effort profiles
