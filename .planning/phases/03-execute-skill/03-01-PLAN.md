**Change 1 -- Reference include path (D-05):**

The current include on line 18 is:
```

```

After moving advisor-timing.md to `references/` at plugin root, the relative path from the skill directory is `../../references/advisor-timing.md`. However, this relative traversal may not work with the `@` include mechanism.

Use this approach: Replace the `@references/advisor-timing.md` include with `@${CLAUDE_PLUGIN_ROOT}/references/advisor-timing.md`. The `${CLAUDE_PLUGIN_ROOT}` variable resolves to the plugin's installation path and is documented for plugin components.

If `${CLAUDE_PLUGIN_ROOT}` is not available in skill bodies (documented for commands but assumed for skills per research A1), the fallback is to instruct the executor to read the file explicitly. In that case, replace the `@references/advisor-timing.md` line with:

Implement the `@${CLAUDE_PLUGIN_ROOT}/references/advisor-timing.md` approach first. The fallback can be applied during Phase 3 execution testing if the include does not resolve.
