<specifics>
## Specific Ideas

- **The MAJOR fix's economic leverage is the plan->execute handoff, not the plan skill alone.** Session 5's 3 advisor consultations happened in execute, not plan, because plan (S1) under-packaged framework details AND execute (S5) had no framework-verification discipline of its own. D-02's symmetric cues in BOTH skills target the handoff failure mode: plan carries framework claims forward via the plan artifact's Pre-Verified Package Behavior Claims section; execute re-verifies them (or gap-fills) against the current codebase before starting substantive work.
- **The 100-word target stays exactly because 80w over-compresses.** Opus 4.7's literal-following makes a 95-100w worked example a stronger lever than a lower abstract target. The smoke-test word-count gate (D-05) is the hard-enforcement lever; the density example (D-04) is the steady-state lever.
- **`DEF-response-structure.sh` keeps its name even though coverage grows.** Renaming a committed smoke test causes ripple churn in docs and CI references. The `DEF-` prefix becomes a historical anchor like "ADR-007" -- it names the original finding set, not the current coverage scope. Future phases adding I, J, K blocks do not rename.
- **Phase 3 mid-execute re-consultation is a feature, not a bug.** The 4 trigger conditions (empirical contradiction, non-trivial approach change, silent failure, strategic shape change) are legitimate escape hatches. Phase 5.5 targets ONE specific subclass of mid-execute re-consultation: those driven by AVOIDABLE framework-convention gaps. The gate at `<=2` for execute is not a ban on Phase 3; it's a target for the happy path, and a manual-inspection trigger when exceeded.
- **Bun-crash on Session 2 is not a Phase 5.5 concern.** The crash is a Windows arm64 + QEMU x86_64 emulation defect in Bun 1.3.13 (panic at `0x7FF71A10355C`). Not a skill defect; not reproducible on other platforms. D-11 explicitly exempts this crash from gate failure -- manual plan-step completion following Phase 5.4 session-notes.md convention is acceptable provided pre-crash metrics pass the gate.
- **Letter continuation G/H for security-reviewer is deliberate.** Phase 5.4 letter convention (A-K for findings) pairs with smoke-test block naming. G = security-reviewer `### Findings` (mirror of F for reviewer). H = security-reviewer `### Threat Patterns` (security-flavored analogue of Cross-Cutting Patterns). This keeps a single smoke-test script readable as "letters cover findings D through H" rather than splitting across scripts.
