### Q2 (clarification): claude -p invocation shape?

**User question:** When you automate the Compodoc UAT, do you use `claude --model sonnet --effort medium -p "<prompt>"`?

**Response:** Core shape yes, with additional flags load-bearing for the measurement pipeline:

```
claude --model sonnet --effort medium --plugin-dir plugins/lz-advisor \
  --dangerously-skip-permissions \
  -p "$(cat "$PROMPT_FILE")" \
  --verbose --output-format stream-json > "$TRACE_FILE" 2>&1
```

| Flag | Why |
|------|-----|
| --model sonnet | CLAUDE.md skill-verification convention + Phase 5.4 baseline |
| --effort medium | Same |
| --plugin-dir plugins/lz-advisor | Resolves /lz-advisor.* skills (not globally installed) |
| --dangerously-skip-permissions | Unblocks advisor Agent spawn |
| -p "$(cat "$PROMPT_FILE")" | Pipes canonical prompt file verbatim |
| --verbose --output-format stream-json | tally.mjs parses stream-json JSONL |
| > "$TRACE_FILE" 2>&1 | Captures full JSONL for Task 4 extraction |

Pre-flight per-session: `head -n 3 "$PROMPT_FILE" | rg -q '^/lz-advisor\.'` hard-fails if missing (T-05.5-15 mitigation).

**User's response:** "Sounds good" -- accepted verbatim.
