## Issues Encountered

1. **RESOLVED — `claude -p` stderr banner in JSONL file.** The invocation `claude -p ... --output-format stream-json > "$TRACE" 2>&1` captures a `Warning: no stdin data received in 3s, proceeding without it` banner on line 1 of every trace (stderr merged with stdout). This causes naive "line 1 is JSON" validation to fail. **Fix:** tally.mjs's existing `try/catch` handles this transparently. Sanity validator updated inline to count parseable JSON lines rather than check line 1 specifically. No file modifications needed; observation only.

2. **RESOLVED — S5 result event `is_error: true`.** The final result event of S5 trace emitted `is_error: true` with `result: "You've hit your org's monthly usage limit"`. **Investigation:** extracted the full result object and traced back to confirm this is a POST-work event; both advisor invocations (turns 28 and 64) and 3 productive commits (`658345d`, `658c2e4`, `ec442e3`) landed on NGX BEFORE the quota hit. Not a skill defect. Documented in session-notes.md and VERIFICATION.md Section 3 "Stage 2 Summary".

3. **RESOLVED — Git Bash `/` path-translation in rg patterns.** Pre-flight audit discovered that `rg -q '^/lz-advisor\.'` returns false-negative under Git Bash on Windows arm64 due to MSYS path-expansion mangling the leading `/`. The bracket-escape form `rg -q '^[/]lz-advisor\.'` is semantically equivalent and avoids the path-expansion. Documented in `preflight-audit.txt` and `session-notes.md`. Prompts are authoritative (direct `head` output confirms all 6 begin with `/lz-advisor.`).
