## Patterns Established

**Shape 3 pattern:** When a `claude -p` invocation's terminal stdout is used as a smoke-test surface, the test implicitly asserts on the FINAL assistant turn only. Any contract that requires content on an intermediate assistant turn (verbatim echo, pre-Write marker, pass-through rendering) becomes invisible to stdout-layer assertions when the final turn is a different text block. The fix is to assert on the `--output-format stream-json` JSONL trace, which preserves all turns.

**JSONL-layer assertion pattern:** For skills that consult an Agent tool, the authoritative content is in the `tool_result.content[0].text` for the matching `tool_use_id`. Use a Node ES module with `JSON.parse` per line (tolerant of partial trailing lines) and two-pass id-binding (first pass: collect tool_use_ids for the target agent; second pass: match tool_result events by tool_use_id). This replaces substring heuristics and is robust across skill-prose drift.

**Smoke-test fixture contamination pattern:** Shell-script smoke tests that reuse a scratch directory across multiple `claude -p` invocations must explicitly clean any stateful artifacts each invocation would produce (e.g., `plans/`, `reviews/`, `db/`). Otherwise the second invocation runs in a re-entrant mode whose advisor/skill behavior diverges from the first-invocation path.
