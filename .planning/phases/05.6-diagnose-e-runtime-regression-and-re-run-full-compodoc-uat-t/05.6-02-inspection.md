## Interpretation

The advisor emitted correct `Assuming X (unverified), do Y.` frames in item 2. Sonnet, after receiving the tool_result, produced a Phase 3 text block (JSONL line 33, assistant message) that verbatim-echoes every numbered item inside properly formed `--- Strategic Direction ---` / `--- End Strategic Direction ---` markers. The echo contract is intact at the JSONL layer.

The regression is therefore not at the Sonnet-paraphrasing layer (Shape 1) and not at the advisor-output layer (Shape 2). Instead, the Phase 3 text block exists correctly inside the stream-json JSONL but did not survive to the terminal stdout that DEF-response-structure.sh reads. The `claude -p` harness is the suspect: the `-p` stdout stream renders only the final result message, not intermediate assistant turns. Since Sonnet's verbatim echo landed in an intermediate assistant message (line 33) while the final result message (line 40) only surfaces the last assistant message's text (line 39, the plan-updated summary), the Strategic Direction block is simply not present in the `-p` stdout even though it was correctly generated.

This is a Shape 3 finding: the JSONL trace preserves the correct Phase 3 behavior, but the harness's stdout rendering of `-p` only exposes the final assistant turn, not intermediate turns. The DEF smoke test is measuring the wrong layer. The `DEF-response-structure.sh` regex applied to stdout is a proxy for Phase 3 correctness that breaks whenever Sonnet emits the echo mid-conversation rather than as the last message before stopping. There is no harness bug to report to Anthropic per se -- the `-p` stdout contract is "final result only" and Sonnet honoring the echo contract mid-conversation is working as intended. The real issue is that DEF is an unreliable signal for the stdout-echo contract.
