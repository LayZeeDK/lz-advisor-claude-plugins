- **Rank 1: Direction D2** -- amend Rule 5b "Format mandate" to differentiate two distinct surfaces:
  - **Internal-prompt surface** (executor -> agent CONTEXT block via Agent tool prompt): canonical `<pre_verified source="..." claim_id="pv-N">...<claim>...</claim><evidence method="...">...</evidence>...</pre_verified>` XML form REQUIRED. Three explicit acceptable shapes: (a) token reference + Verified: trailer in commit bodies; (b) token reference + prose citation in plan/review/security-review bodies; (c) inline parenthetical token reference in finding lines. Forbidden plain-bullet pattern preserved.

- **Reject Direction D1** (tighten enforcement -- require XML blocks in user-facing artifacts too): fights documentation convention; Anthropic XML guidance is for prompt INPUTS not OUTPUTS; markdown plan files + commit bodies + review output are human-readable artifacts; embedding XML in markdown bodies makes them harder to read AND fights Sonnet 4.6 / Opus 4.7 markdown rendering training. Per Plan 07-09's empirical lesson, fighting the model's training is the worst tradeoff.

- **Reject Direction D3** (status quo + doc note): doesn't amend the rule; creates a contradiction (rule says one thing, note says another); doesn't enable Assertion 6 structural integrity check.
