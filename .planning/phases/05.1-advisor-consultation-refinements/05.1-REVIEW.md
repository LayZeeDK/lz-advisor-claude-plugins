## Summary

This changeset refines advisor consultation patterns across the lz-advisor plugin: adding "Final Response Discipline" to all three agents, replacing the advisor's clarification-request behavior with conditional recommendations (critical given `maxTurns: 1`), adding "What to Include Verbatim" guidance to plan and execute skills, expanding mid-execution re-consultation criteria, and adding web content verbatim guidance to advisor-timing.md.

The changes are well-motivated and internally consistent across most files. Two warnings relate to contradictory guidance within the execute skill that could cause the executor to package incomplete advisor prompts. Two informational items note duplication and a minor tension in the reference file.

## Warnings

### WR-01: Contradictory Brevity vs Self-Containment Guidance in Execute Skill

**File:** `plugins/lz-advisor/skills/execute/SKILL.md:118-121` vs `plugins/lz-advisor/skills/execute/SKILL.md:131-138`
**Issue:** Lines 118-121 instruct the executor to "Keep re-consultation prompts brief (2-3 sentences)" and state these calls "reuse the pre-execute consultation context implicitly." Lines 131-138 then say "Each advisor call must be self-contained" and list 4 required elements (original task, progress summary, current problem, specific question). These two directives conflict: a self-contained prompt with 4 required elements cannot fit in 2-3 sentences, and the advisor does not retain context between calls (`maxTurns: 1`, fresh context each time per advisor-timing.md line 46). The phrase "reuse the pre-execute consultation context implicitly" is misleading -- the advisor has no memory of prior calls.

An executor following the brevity guidance would send terse prompts missing context the single-turn advisor needs. An executor following the self-containment guidance would ignore the 2-3 sentence target.

**Fix:** Reconcile by adjusting lines 118-121. Remove the "reuse context implicitly" claim and adjust the brevity target to match what self-containment requires. For example:

```markdown
Keep re-consultation prompts focused: include the 4 required elements
from Context Packaging below, but keep each element to 1-2 sentences.
The advisor starts fresh each call -- all relevant context must be in
the prompt, but summaries are sufficient for elements 1-2 when the
current problem (element 3) is the primary focus.
```

### WR-02: advisor-timing.md Has Contradictory Verbatim vs Summarize Guidance

**File:** `plugins/lz-advisor/references/advisor-timing.md:57-64`
**Issue:** Lines 57-59 say "Summarize findings; do not paste entire files." Lines 61-64 say web content "should be included in the advisor prompt verbatim." These two directives sit in the same section without the disambiguation that the skill files provide. The skills' "What to Include Verbatim" subsections resolve this tension by distinguishing source material (verbatim) from executor findings (summarize) and adding a 2,000-word threshold. But since advisor-timing.md is referenced by all 4 skills via `@${CLAUDE_PLUGIN_ROOT}/references/advisor-timing.md`, an executor reading only the reference file encounters the contradiction without resolution.

**Fix:** Add the disambiguation inline in advisor-timing.md, aligning with the skills' guidance:

```markdown
Keep the prompt focused and summarized. The advisor works best with clear,
organized context -- not raw file dumps or verbose tool output. Summarize
your own orientation findings; do not paste entire files.

Web content the executor pre-fetches during orientation (documentation pages,
API references, release notes) should be included verbatim -- the advisor has
no network access and cannot retrieve content you omit. If user-pasted
material exceeds roughly 2,000 words, quote the sections relevant to the
decision rather than including the full document.
```

## Info

### IN-01: Identical "Final Response Discipline" Section Across All 3 Agents

**File:** `plugins/lz-advisor/agents/advisor.md:99-113`, `plugins/lz-advisor/agents/reviewer.md:101-115`, `plugins/lz-advisor/agents/security-reviewer.md:113-127`
**Issue:** The exact same 12-line "Final Response Discipline" block is duplicated verbatim across all three agent files. This is structurally necessary because agent markdown files are standalone (they cannot include shared references), but it creates a maintenance burden -- any update must be applied to all three files.
**Fix:** No code change needed now; this is a known limitation of the agent file format. If the text ever needs updating, ensure all three files are modified in lockstep. Consider adding a comment in each agent file (e.g., `<!-- Shared: keep in sync with advisor.md, reviewer.md, security-reviewer.md -->`) to flag the coupling.

### IN-02: Identical "What to Include Verbatim" Section in Plan and Execute Skills

**File:** `plugins/lz-advisor/skills/plan/SKILL.md:59-75`, `plugins/lz-advisor/skills/execute/SKILL.md:70-86`
**Issue:** The exact same 16-line block appears in both skills. Unlike agents, skills can reference shared content via `@${CLAUDE_PLUGIN_ROOT}/references/`. This content could be consolidated into `advisor-timing.md` (which both skills already reference) to eliminate duplication.
**Fix:** Move the "What to Include Verbatim" content into `references/advisor-timing.md` under the "What to Include in the Advisor Prompt" section, then remove the duplicated subsections from both skills. The `@` reference already loads the file, so the guidance would be available without duplication.
