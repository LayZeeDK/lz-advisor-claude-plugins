# Quick Task 260611-f98: Address the review findings (3rd /code-review pass) Summary

Convergence-round fix for the 3 surviving findings on PR #1: collapsed the per-finding-validation (PFV) accumulator to two `case` arms in both budget fixtures (closing the relocated multi-paragraph under-count gap and deleting an unreachable boundary arm), and added a short clause to the security-reviewer holistic Threat Patterns synthesis acknowledging the now-confirmed Finding 3 (A05 CORS prod-config).

## What Was Built

### Task 1 -- FIX #1 + FIX #2 (commit 62955ab)

Both budget fixtures (`tests/D-reviewer-budget.sh`, `tests/D-security-reviewer-budget.sh`) carry a byte-identical PFV accumulator loop. Applied the same two deletions to each:

- **FIX #1**: deleted the `""` blank-line boundary `case` arm. A blank line now falls through to the `*)` continuation arm; `wc -w` of an empty line is 0, so intra-entry blanks contribute nothing and no longer close an entry. This closes the relocated under-count gap: a single logical PFV entry written as two blockquote paragraphs (opening line + blank + continuation, no second `Validation of Finding N:` prefix) is now counted whole instead of counting only the first paragraph.
- **FIX #2**: deleted the unreachable `"### "*)` boundary arm. `extract_section()` already stops at -- and never emits -- the next `### ` heading into `PFV_BODY`, so the arm could never fire; after FIX #1 narrowed the finalizers it was doubly redundant.

The resulting loop has exactly two arms: `"Validation of Finding "*)` (finalize previous open entry, start new -- unchanged) and `*)` (accumulate words when `pfv_in_entry -eq 1` -- unchanged). The pre-loop comment block was reworded to state that intra-entry blanks are continuation and only `Validation of Finding ` + the EOF flush finalize. Constants (`PFV_CAP=60`, `FINDING_RE`, `SEV_HEADERS`), `finalize_pfv_entry`, the post-loop EOF flush, the present-but-unparsed loud-fail, and the 4-variable state are all unchanged. Fixtures remain standalone (D-10, no dedup).

### Task 2 -- FIX #3 (commit 1a46b93)

`plugins/lz-advisor/agents/security-reviewer.md` line 245 (the blockquoted holistic `### Threat Patterns` synthesis) previously enumerated findings 1, 4, 2, 6, and 5 but omitted finding 3.
